Direct Memory Access (DMA) is a specialized data transfer technique designed to bypass the CPU, allowing peripheral devices to communicate directly with main memory to overcome the inherent inefficiencies of programmed and interrupt-driven I/O. The process is initiated when the CPU issues an I/O command to the DMA controller, providing essential parameters such as the device port number, the destination memory address, the data count, and specific control signals. Following this instruction, a handshaking sequence occurs in which the DMA controller requests control of the system bus via a "Hold" signal. Once the CPU grants a "Hold Acknowledgement," the DMA controller assumes command of the bus, enabling autonomous data transfer while the processor is liberated to perform other computational tasks.

The efficiency of this data exchange is dictated by the transfer mode employed by the controller. In cycle stealing mode, data is moved byte-by-byte or word-by-word by utilizing single, intermittent bus cycles. Conversely, burst mode—also referred to as bulk or batch mode—leverages high-capacity buffers to facilitate the continuous transfer of large data blocks. Regardless of the mode, the process involves moving data from an I/O device into a temporary buffer before loading it onto the system bus for the main memory to read, a cycle that repeats until the internal word counter reaches zero, signifying the completion of the transfer without further processor intervention.

Burst mode operations are further defined by two distinct phases: preparation time and transfer time. During the preparation phase, the slower disk device fills the DMA buffer byte-by-byte; because the system bus remains free during this period, the CPU can continue executing other tasks. However, during the transfer phase, the DMA controller takes exclusive control of the system bus to move the entire buffer contents to memory, placing the CPU in a "blocked state." Mastery of these timing cycles and the resulting CPU states is essential for calculating system efficiency and solving complex architectural problems related to data transfer overhead in high-speed environments.
