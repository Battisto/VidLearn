Direct Memory Access (DMA) is a specialized data transfer technique designed to enhance system efficiency by allowing peripheral devices to communicate directly with main memory, thereby bypassing the central processing unit (CPU). The process begins when the CPU initiates an I/O command by providing the DMA controller with essential parameters, including the specific port number, the target memory address, and the total count of data to be transferred. By offloading these data-intensive tasks to a dedicated controller, the CPU is freed to execute other instructions, significantly optimizing the computer's overall processing power.

To manage the transfer independently, the DMA controller must first gain control of the system bus. Once the I/O device signals its readiness, the controller issues a HOLD request to the CPU. Upon receiving a hold acknowledgment, the CPU relinquishes the bus, allowing the DMA controller to manage the movement of data from the peripheral into a local buffer. Once this buffer is filled, the data is placed onto the system bus for the main memory to read. The efficiency of this exchange is largely determined by the buffer size and the specific operational mode employed, typically categorized as cycle stealing or burst mode.

Cycle stealing involves the incremental transfer of data, where the controller "steals" individual bus cycles to move data byte-by-byte or word-by-word. In contrast, burst mode—often referred to as bulk or batch mode—utilizes larger buffers to facilitate the high-speed transfer of entire data blocks. This mode is defined by two distinct phases: preparation time and transfer time. During the preparation phase, the disk fills the DMA buffer while the CPU remains in a "busy state," continuing to execute unrelated tasks. However, during the transfer phase, the buffer occupies the system bus to move data to memory, forcing the CPU into a "blocked state." Distinguishing between these two timings is essential for calculating bus utilization and assessing the total efficiency of DMA operations.
